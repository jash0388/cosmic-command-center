# DataNauts

A modern remote desktop / device control SaaS — connect, monitor and control your PC, Mac, iOS and Android devices from a single dashboard. Built with TanStack Start, React, TypeScript, Tailwind v4, Zustand, React Query, and Lovable Cloud (Supabase under the hood) for auth, database and realtime.

---

## ✨ What's inside

### Landing site (public)
- `/` — Hero, Features, How it works, Pricing, FAQ, Download, Final CTA
- `/download` — Download page for Win / Mac / iOS / Android
- `/legal/privacy`, `/legal/terms` — Legal pages
- `/auth/login`, `/auth/register` — Email + password auth

### Dashboard (authenticated)
- `/dashboard` — Sidebar app with:
  - **Overview** — KPIs, live device list, recent sessions
  - **Devices** — Full CRUD: add, rename, delete, search, status badges
  - **Activity** — Session history with duration & latency
  - **Settings** — Profile / account
  - **Quick Connect** — 8-char pairing code with 30s rotation

### Design system
- Dark theme with white text, multi-color accents (orange, pink, purple, blue, cyan, green, yellow)
- Tailwind v4 via `@import "tailwindcss"` in `src/styles.css`
- Semantic tokens (`--bg`, `--surface`, `--text`, `--orange`, `--purple`, …) — never hardcode colors in components
- Snappy button transitions (~80ms linear) so clicks feel instant
- Fonts: DM Sans (UI) + JetBrains Mono (eyebrows / numbers)

## 🗄️ Database schema

Migrations live in `supabase/migrations/`. The current schema:

### `profiles`
| column | type | notes |
|---|---|---|
| `id` | `uuid` PK | matches `auth.users.id` |
| `display_name` | `text` | from signup metadata or email prefix |
| `created_at` | `timestamptz` | default `now()` |

Auto-populated by trigger `handle_new_user()` on `auth.users` insert.

### `devices`
| column | type | notes |
|---|---|---|
| `id` | `uuid` PK | |
| `user_id` | `uuid` | owner |
| `name` | `text` | e.g. "Main Workstation" |
| `type` | enum `device_type` | `desktop` / `laptop` / `mobile` / `tablet` |
| `os` | `text` | e.g. "macOS 14" |
| `status` | enum `device_status` | `online` / `idle` / `offline` |
| `latency_ms` | `int` | last measured RTT |
| `last_seen_at` | `timestamptz` | |

### `sessions`
| column | type | notes |
|---|---|---|
| `id` | `uuid` PK | |
| `user_id` | `uuid` | |
| `device_id` | `uuid` | |
| `started_at` | `timestamptz` | |
| `ended_at` | `timestamptz` | nullable while live |
| `duration_seconds` | `int` | |
| `latency_ms` | `int` | |

### `pairing_codes`
| column | type | notes |
|---|---|---|
| `id` | `uuid` PK | |
| `user_id` | `uuid` | |
| `code` | `text` | 8-char rotating code |
| `expires_at` | `timestamptz` | TTL ~30s |
| `claimed` | `bool` | |

### Row Level Security
RLS is **enabled on every table**. Policies restrict `select / insert / update / delete` to rows where `auth.uid() = user_id` (or `= id` for `profiles`). Users only ever see their own data.

## 🚀 Run locally

```bash
# 1. install deps
bun install        # or: npm install / pnpm install

# 2. environment (.env is auto-generated by Lovable Cloud — see below if self-hosting)
#    VITE_SUPABASE_URL=...
#    VITE_SUPABASE_PUBLISHABLE_KEY=...

```

Open http://localhost:5173, register at `/auth/register`, then jump to `/dashboard`.

## 🌐 Deploying

### Option A — One-click via Lovable (recommended)
1. Open the project in Lovable.
2. Click **Publish** (top-right). This builds and ships the frontend to a `*.lovable.app` URL.
3. Backend (DB, auth, RLS) deploys **automatically** — no extra step.
4. Optional: connect a custom domain from **Project Settings → Domains**.

Frontend changes need a re-publish. Database / auth changes go live immediately.

### Option B — GitHub + your own host
You can push the project to GitHub (Lovable → GitHub integration) and deploy the built output anywhere that supports Node / edge runtimes (Vercel, Netlify, Cloudflare Pages, Fly, your own VPS).

```bash
# build
bun run build

# start production server
bun run start
```

#### Required environment variables in your host
| Var | Where used | Public? |
|---|---|---|
| `VITE_SUPABASE_URL` | browser client | ✅ yes |
| `VITE_SUPABASE_PUBLISHABLE_KEY` | browser client | ✅ yes (anon key) |
| `VITE_SUPABASE_PROJECT_ID` | browser client | ✅ yes |
| `SUPABASE_URL` | server functions | ✅ yes |
| `SUPABASE_PUBLISHABLE_KEY` | server functions | ✅ yes |
| `SUPABASE_SERVICE_ROLE_KEY` | server admin ops only | 🚫 **secret — never expose** |

#### Self-hosting Supabase (if not using Lovable Cloud)
1. Create a Supabase project at https://supabase.com.
2. From `supabase/migrations/`, run each `.sql` file in order via the Supabase SQL editor (or `supabase db push` if using the CLI).
3. In **Authentication → Providers**, enable **Email**. Decide whether to require email confirmation (default in Supabase = on; the Lovable dev project uses auto-confirm).
4. Copy `Project URL`, `anon key`, `service_role key` from **Settings → API** into your host's env vars (mapped to the table above).
5. Re-deploy.

## 🧩 Adding features

- **New page** → drop a file in `src/routes/`, e.g. `src/routes/billing.tsx` with `createFileRoute("/billing")`. The router regenerates automatically.
- **New table** → write a migration in `supabase/migrations/`, always with RLS policies scoped to `auth.uid()`.
- **Server-side logic** → use `createServerFn` from `@tanstack/react-start` (NOT Supabase Edge Functions).
- **New color** → add a token in `src/styles.css` under `:root` and reference it via `var(--…)` or a utility class.

## 🔐 Security checklist

- ✅ RLS enabled on all user tables
- ✅ Service role key only used server-side
- ✅ Auth state listened to globally (`AuthProvider`) so React Query caches invalidate on login/logout
- ✅ No secrets in client bundle (`VITE_*` only)
